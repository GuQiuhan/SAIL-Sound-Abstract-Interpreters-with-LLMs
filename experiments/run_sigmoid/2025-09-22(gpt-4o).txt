When the operator takes multiple inputs, use `prev_0`, `prev_1`, ... to refer to each input.

This transformer follows standard DeepPoly overapproximation for the sigmoid activation function:

If prev[u] ≤ 0: the input is entirely on the left side of the sigmoid curve, where it behaves like a convex function approaching 0 → approximated with a line from (prev[l], σ(prev[l])) to (prev[u], σ(prev[u])), lower and upper bounds are tight.

If prev[l] ≥ 0: the input is entirely on the right, where sigmoid is concave and upper bound is tighter.

If input straddles zero: we use a conservative approximation covering the full range [0,1] with simple constant bounds.
